# InvestSkill 使用说明

## 快速了解

InvestSkill 是一套**专业美股投资分析框架**，包含 20 个结构化 prompt 模板。  
它不是传统软件——无需安装依赖、无需 API Key、无需联网——通过读取 prompt 文件，即可让 AI 按照专业投资分析师的标准流程来分析美股。

- **版本**: v1.6.0 (当前仓库)
- **许可**: MIT 开源
- **作者**: [yennanliu](https://github.com/yennanliu)
- **本地位置**: `/home/severin/Codelib/股市分析/InvestSkill/`

---

## 安装方式

InvestSkill 已 clone 到本地并在 OpenCode 注册为 skill，无需额外操作。

```
~/.config/opencode/skills/invest-skill/SKILL.md   ← OpenCode skill 注册文件
```

---

## 使用方法

### 方式一：直接要求（推荐）

直接告诉 AI 你想分析什么，AI 会自动加载 `invest-skill` skill 并调用对应框架：

```
帮我全面分析 AAPL
对 NVDA 做一个 DCF 估值
解读这份微软的 10-K 报告
```

### 方式二：指定 prompt 文件

你可以明确指定使用哪个分析框架：

```
请读取 /home/severin/Codelib/股市分析/InvestSkill/prompts/stock-eval.md
然后分析 TSLA
```

```
请按 dcf-valuation.md 的框架给 MSFT 做个估值
要求：三种场景（乐观/基准/悲观）
```

### 方式三：粘贴数据后分析

部分框架需要你提供原始数据（如财报、电话会议记录）：

```
请读取 financial-report-analyst.md
然后分析以下内容：[粘贴 10-K 原文]
```

## 20 个分析框架速查

### 📊 核心分析（日常最常用）

| 命令 | 用途 | 输入 |
|---|---|---|
| `stock-eval AAPL` | 一键综合评估：财务健康、估值、护城河、风险矩阵 | 股票代码 |
| `fundamental-analysis MSFT` | 深度财报分析（加 `--visual` 可视化输出） | 股票代码 |
| `technical-analysis TSLA` | 技术面：趋势、量价分布、Ichimoku 云图（加 `--chart`） | 股票代码 |
| `economics-analysis` | 宏观经济：收益率曲线、信用利差、衰退概率评分 | 无（当前宏观数据） |
| `portfolio-review` | 检视你的持仓组合，给出优化建议 | 粘贴持仓清单 |
| `sector-analysis` | 行业定位与轮动分析 | 无或指定行业 |

### 💰 估值（判断贵贱）

| 命令 | 用途 |
|---|---|
| `dcf-valuation AAPL` | DCF 模型：3 场景敏感性分析 + 安全边际 |
| `stock-valuation MSFT` | 多方法估值汇总：DCF + 可比公司 + EV/EBITDA + P/E + 剩余收益，足球场图 |

### 📄 财报解读

| 命令 | 用途 | 输入 |
|---|---|---|
| `financial-report-analyst AAPL 10-K` | 年报深度解读：会计质量、MD&A、红旗检测 | 股票代码 + 文件类型 |
| `earnings-call-analysis` | 财报电话会议：情绪分析、管理层态度、风险提示 | 粘贴会议记录 |

### 🔍 市场监控

| 命令 | 用途 |
|---|---|
| `insider-trading AAPL` | SEC Form 4 内部人买卖追踪 |
| `institutional-ownership MSFT` | 13F 机构持仓变化 |
| `dividend-analysis KO` | 股息安全：Chowder Rule、收益率陷阱 |
| `short-interest TSLA` | 空头挤压评分、借券利率 |

### 🧠 高级分析

| 命令 | 用途 |
|---|---|
| `competitor-analysis AMD --peers NVDA,INTC` | 护城河评分 + 波特五力 + 竞争对标 |
| `options-analysis AAPL` | 期权希腊字母、IV 排名、策略推荐 |
| `chart-master` | 图表模式识别专家 |
| `result-validator` | 交叉检验分析结果的可靠性 |

### 🔄 元框架（终极武器）

| 命令 | 用途 |
|---|---|
| `research-bundle AAPL` | **全流程链式分析**：串联所有模块 → 统一投资论点 + 综合评分 |
| `full-report AAPL` | 生成完整的投资研究报告 |

## 常见场景速查表

| 你想知道... | 用哪个 |
|---|---|
| "这只股票值不值得买？" | `stock-eval` + `stock-valuation` |
| "内在价值是多少？" | `dcf-valuation` |
| "财报有没有雷？" | `financial-report-analyst` |
| "高管在增持还是减持？" | `insider-trading` |
| "股息还能拿多久？" | `dividend-analysis` |
| "技术面支撑/压力在哪？" | `technical-analysis` |
| "和同行比怎么样？" | `competitor-analysis` |
| "要不要做期权策略？" | `options-analysis` |
| "全面深度尽调" | `research-bundle` ← 最强 |

## 输出格式

每个分析框架结尾都会输出标准化的 **Investment Signal Block**：

```
┌─────────────────────────────────────┐
│  Signal: 🟢 BULLISH                  │
│  Confidence: HIGH                    │
│  Target Price: $195                  │
│  Key Risk: 监管风险                  │
│  Key Catalyst: 新产品发布            │
└─────────────────────────────────────┘
```

## 注意事项

1. **仅支持美股**：框架基于 SEC 文件、美股财务指标设计
2. **无需外部 API**：纯 prompt 框架，分析依赖 AI 的知识库和搜索能力
3. **数据时效性**：AI 可能无法获取实时价格/财报数据，建议先获取最新数据再分析
4. **仅供研究参考**：所有分析不构成投资建议
